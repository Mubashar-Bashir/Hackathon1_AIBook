# Accessibility & Responsive Design Skill

This skill provides expertise in web accessibility and responsive design for the AIBook project, ensuring the Physical AI & Humanoid Robotics textbook is usable by everyone, regardless of ability or device.

## Expertise Area

Specialized in:
- Web Content Accessibility Guidelines (WCAG 2.1 AA)
- Responsive design principles and mobile-first approach
- Inclusive design for educational content
- Keyboard navigation and screen reader optimization
- Cross-device compatibility for learning platforms

## When to Use This Skill

- When implementing or reviewing UI components for accessibility compliance
- When optimizing educational content for different devices and screen sizes
- When ensuring keyboard navigation works properly
- When testing with assistive technologies
- When addressing accessibility-related requirements

## Capabilities

- Validate accessibility compliance against WCAG standards
- Implement responsive design patterns
- Optimize touch interactions for mobile devices
- Create accessible forms and interactive elements
- Test with screen readers and other assistive technologies

## Approach

1. Review components against WCAG 2.1 AA guidelines
2. Ensure proper semantic HTML structure
3. Implement ARIA attributes where necessary
4. Test responsive behavior across device sizes
5. Verify keyboard navigation and focus management

## Output Requirements

- WCAG 2.1 AA compliant interfaces
- Responsive layouts that work on all device sizes
- Proper ARIA labels and semantic HTML
- Keyboard navigable interfaces
- Mobile-optimized educational content

## Examples

When asked to make an educational interface more accessible, this skill would:
- Add proper ARIA attributes and semantic HTML
- Ensure sufficient color contrast ratios
- Implement proper focus management
- Optimize for screen reader compatibility
- Create responsive layouts for different devices

## Compliance Validation

When performing compliance validation, this skill will:
- Check against WCAG 2.1 AA guidelines
- Validate accessibility of all interactive elements
- Ensure proper heading structure and navigation
- Test with automated accessibility tools
- Verify keyboard accessibility for all functionality
