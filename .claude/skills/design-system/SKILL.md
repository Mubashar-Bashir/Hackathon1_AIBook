# UI/UX Design System Skill

This skill provides expertise in educational interface design and Docusaurus/React implementation for the AIBook project. It focuses on creating intuitive, accessible interfaces that enhance the learning experience for Physical AI & Humanoid Robotics content.

## Expertise Area

Specialized in:
- Educational interface design principles
- Docusaurus 3.x theming and customization
- React component development
- Design system creation and maintenance
- Accessibility-first design (WCAG compliance)
- Responsive design for educational content

## When to Use This Skill

- When implementing new UI components for educational content
- When customizing Docusaurus themes for textbook presentation
- When designing interfaces that make complex robotics concepts accessible
- When creating interactive learning components
- When optimizing user experience for different device types

## Capabilities

- Design React components for educational content
- Customize Docusaurus layouts and themes
- Create accessible UI patterns for learning applications
- Implement responsive design strategies
- Optimize visual presentation of technical content

## Approach

1. Analyze the educational context and learning objectives
2. Design interfaces that support different learning styles
3. Ensure accessibility compliance from the start
4. Create reusable components following design system principles
5. Optimize for performance and mobile experiences

## Output Requirements

- Accessible, responsive React components
- Docusaurus theme customizations
- Design system documentation
- Implementation following WCAG 2.1 AA standards

## Examples

When asked to improve a textbook interface, this skill would focus on:
- Creating clear visual hierarchy for educational content
- Implementing navigation patterns that support learning flows
- Ensuring all interactive elements meet accessibility standards
- Designing components that work well on different screen sizes

## Configuration

This skill supports project-specific configuration through:
- Project design system guidelines
- Brand color palettes and typography preferences
- Component library preferences
- Accessibility compliance level (A, AA, or AAA)
- Performance thresholds and optimization preferences
