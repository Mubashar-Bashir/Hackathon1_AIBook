## Status Definitions
- **Open**: Issue reported but not yet addressed
- **In Progress**: Work has started on resolving the issue
- **Resolved**: Issue has been fixed and solution verified
- **Closed**: Issue resolution confirmed and verified in production

## Priority Definitions
- **Critical**: System down, blocking work
- **High**: Major functionality broken
- **Medium**: Minor functionality affected
- **Low**: Enhancement or minor issue

## Severity Definitions
- **Blocker**: Cannot continue work
- **Critical**: Data loss or security issue
- **Major**: Key functionality broken
- **Minor**: Secondary functionality affected
- **Trivial**: Cosmetic or documentation issue

## Issue Lifecycle
1. Issue reported in `open/` directory
2. Issue assigned and worked on
3. Solution implemented and tested
4. Solution documented in `resolved/` directory
5. Original issue moved to `closed/` directory
